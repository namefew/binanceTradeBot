## 🚀 批量回测功能

### 功能说明
**一次加载数据，测试所有策略**，避免重复请求币安 API，节省时间并减少失败率。

### 使用方法

1. **打开 Web 界面**
   ```bash
   streamlit run app.py
   ```

2. **选择「🚀 批量回测」标签页**

3. **配置参数**
   - 📊 回测天数：建议 30-90 天
   - 💰 初始资金：默认 $10,000
   - 📈 仓位比例：默认 10%

4. **点击「🚀 开始批量回测（所有策略）」按钮**

5. **等待完成**
   - 系统会依次测试所有 140+ 个策略
   - 进度条显示当前进度
   - 自动保存结果

### 结果展示

#### 📊 策略对比表格
- 按收益率排序
- 前3名高亮显示（绿色）
- 亏损策略标记（红色）

#### 📈 统计分析
- 盈利/亏损策略数量
- 平均收益率
- 最佳策略

#### 📉 可视化图表
- **收益率对比图**：柱状图展示各策略收益
- **交易频率 vs 收益**：散点图分析交易次数与收益的关系

### 优势
✅ **只加载一次数据**：避免频繁请求 API 导致失败  
✅ **全面对比**：一次性看到所有策略表现  
✅ **快速筛选**：找出最适合当前市场的策略  
✅ **可视化分析**：直观的图表帮助决策

## 🔍 策略追踪功能

### 功能说明
选择特定策略和交易对进行**长期模拟运行**，持续记录表现，方便追踪几天或更长时间的结果。

### 使用方法

1. **打开 Web 界面**
   ```bash
   streamlit run app.py
   ```

2. **选择「🔍 策略追踪」标签页**

3. **配置追踪参数**
   - 🎯 **选择策略**：从下拉框选择要追踪的策略
   - 💹 **选择交易对**：支持所有 USDT 交易对（400+）
   - ⏱️ **时间周期**：1m, 5m, 15m, 1h, 4h, 1d
   - 📅 **回测天数**：7-180 天
   - 💰 **初始资金**：默认 $10,000

4. **追踪设置**
   - ✅ **自动刷新**：每 5 分钟自动更新数据
   - ✅ **保存历史记录**：自动保存到 `tracking_history.json`

5. **点击「▶️ 开始追踪」按钮**

### 结果展示

#### 📊 实时绩效指标
- 总收益率
- 最大回撤
- 夏普比率
- 交易次数

#### 💹 资金曲线
动态展示资金变化趋势

#### 📈 历史追踪对比
如果启用了"保存历史记录"，会显示：
- **历史数据表格**：每次追踪的详细记录
- **收益趋势图**：展示策略表现的演变

#### 📋 所有追踪历史
页面底部显示所有策略的追踪历史：
- 支持按策略名称过滤
- 支持按交易对过滤
- 可清空历史记录

### 应用场景

#### 场景1: 策略验证
```
目标：验证某个策略在 BTCUSDT 上的长期表现

步骤：
1. 选择策略：Moving Average Strategy
2. 选择交易对：BTCUSDT
3. 时间周期：1h
4. 回测天数：30天
5. 启用"保存历史记录"
6. 每天运行一次，持续一周
7. 查看历史对比，分析稳定性
```

#### 场景2: 多策略对比
```
目标：对比多个策略在同一交易对上的表现

步骤：
1. 分别追踪 3-5 个感兴趣的策略
2. 使用相同的交易对和参数
3. 连续追踪几天
4. 在"所有追踪历史"中对比结果
5. 选择表现最稳定的策略
```

#### 场景3: 市场适应性测试
```
目标：测试策略在不同市场环境下的表现

步骤：
1. 选择一个策略
2. 分别在牛市、熊市、震荡市期间追踪
3. 对比不同时期的表现
4. 评估策略的适应性
```

## ❓ 常见问题

### Q1: 批量回测很慢怎么办？
**A:** 
- 减少回测天数（建议 30-60 天）
- 使用较短的时间周期（如 1h 而不是 1m）
- 这是正常的，因为要测试 140+ 个策略

### Q2: 为什么有些策略显示错误？
**A:** 
- 某些策略可能需要特定的数据格式
- 可能是策略本身的 bug
- 查看错误信息，必要时调整参数

### Q3: 策略追踪的"自动刷新"怎么用？
**A:** 
- 勾选"自动刷新"后，页面会在 5 分钟后自动重新运行
- 适合长时间监控策略表现
- 注意：这会阻塞浏览器，建议不勾选，手动定期刷新

### Q4: 如何清空追踪历史？
**A:** 
- 在"所有追踪历史"部分，点击"🗑️ 清空所有历史记录"按钮
- 或直接删除 `tracking_history.json` 文件

### Q5: 可以导出追踪结果吗？
**A:** 
- 目前数据保存在 JSON 文件中
- 可以用 Python/pandas 读取并转换为 Excel/CSV
- 示例代码：
  ```python
  import pandas as pd
  df = pd.read_json('tracking_history.json')
  df.to_excel('tracking_results.xlsx', index=False)
  ```

### Q6: 批量回测和策略追踪有什么区别？
**A:** 

| 功能 | 批量回测 | 策略追踪 |
|------|---------|---------|
| **目的** | 快速对比所有策略 | 长期监控特定策略 |
| **策略数量** | 所有 140+ 个 | 单个策略 |
| **运行频率** | 一次性 | 可多次运行 |
| **数据加载** | 只加载一次 | 每次独立加载 |
| **历史记录** | 不保存 | 自动保存 |
| **适用场景** | 策略筛选 | 策略验证 |

## 💡 使用建议

### 新手入门流程

1. **第一步：批量回测**
   - 选择主流交易对（BTCUSDT, ETHUSDT）
   - 回测 30-60 天
   - 找出收益率最高的前 10 个策略

2. **第二步：策略追踪**
   - 从前 10 个策略中选择 2-3 个
   - 分别追踪 7-14 天
   - 观察稳定性和一致性

3. **第三步：实盘测试**
   - 选择表现最好的策略
   - 用小资金实盘测试
   - 持续监控和调整

### 高级技巧

#### 1. 多时间周期验证
```
同一策略在不同时间周期上的表现可能差异很大：
- 短线：1m, 5m, 15m
- 中线：1h, 4h
- 长线：1d

建议：在多个时间周期上测试，选择最稳定的
```

#### 2. 多交易对验证
```
策略可能在某些交易对上表现好，在其他交易对上表现差：
- 主流币：BTCUSDT, ETHUSDT, BNBUSDT
- 山寨币：SOLUSDT, ADAUSDT, DOTUSDT

建议：至少在 3-5 个不同交易对上测试
```

#### 3. 市场环境分析
```
记录每次追踪时的市场环境：
- 牛市：价格上涨趋势
- 熊市：价格下跌趋势
- 震荡市：横盘整理

建议：分析策略在不同市场环境下的表现
```
